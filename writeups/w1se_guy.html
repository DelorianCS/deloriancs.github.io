W1seGuy

Enumeration

This room challenges us to break a weak XOR encryption scheme. When connecting to the server, a flag is encrypted with a random 5-character key. The key loops over the plaintext.

We know TryHackMe flags always start with THM{, which gives us an advantage: a known-plaintext XOR attack. From that, we can recover part of the key and brute-force the rest.

Exploitation

We start by XORing the encrypted bytes against THM{ to extract a partial key. Then we brute-force the remaining 1 character. Below is a clean Python script built to automate the entire process.

import argparse
import itertools
import string
from concurrent.futures import ThreadPoolExecutor, as_completed

def recover_prefix(cipher_hex, known_plain):
    data = bytes.fromhex(cipher_hex)
    prefix = ""
    for i in range(min(len(data), len(known_plain))):
        prefix += chr(data[i] ^ ord(known_plain[i]))
    return prefix

def xor_decrypt(cipher_hex, key):
    encrypted = bytes.fromhex(cipher_hex)
    return ''.join(chr(b ^ ord(key[i % len(key)])) for i, b in enumerate(encrypted))

def verify_key(candidate, cipher_hex, start_text, end_text):
    output = xor_decrypt(cipher_hex, candidate)
    if output.startswith(start_text) and output.endswith(end_text):
        return candidate, output
    return None

def brute_force(cipher_hex, prefix, start_text, end_text):
    charset = string.ascii_letters + string.digits
    remain = 5 - len(prefix)
    results = []

    with ThreadPoolExecutor(max_workers=8) as executor:
        tasks = [
            executor.submit(verify_key, prefix + ''.join(p), cipher_hex, start_text, end_text)
            for p in itertools.product(charset, repeat=remain)
        ]
        for task in as_completed(tasks):
            res = task.result()
            if res:
                results.append(res)

    return results

def main():
    parser = argparse.ArgumentParser(description="Break XOR-encrypted TryHackMe flag")
    parser.add_argument("-e", "--encrypted", required=True, help="Hex string of encrypted flag")
    args = parser.parse_args()

    known_start = "THM{"
    known_end = "}"

    prefix = recover_prefix(args.encrypted, known_start)
    print(f"[*] Extracted key prefix: {prefix}")

    matches = brute_force(args.encrypted, prefix, known_start, known_end)

    if matches:
        for k, flag in matches:
            print(f"[+] Key found: {k}")
            print(f"[+] Decrypted flag: {flag}")
    else:
        print("[-] No valid key found.")

if __name__ == "__main__":
    main()

This script demonstrates how known-plaintext attacks can break XOR with short keys. Try it against the encrypted output you get from the W1seGuy challenge!

Takeaways

  • Always assume short keys with repeating patterns are weak
  • XOR is reversible — and predictable when the plaintext is partially known
  • TryHackMe flags give a reliable prefix and suffix for brute-force validation
  • CyberChef is great for visualizing XOR, but scripting gives full control
